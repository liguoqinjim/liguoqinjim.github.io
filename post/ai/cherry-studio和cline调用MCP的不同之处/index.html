Cherry Studio和cline调用MCP的不同之处

  • Cherry Studio使用模型的Function Call来实现MCP的调用。

    • 所以在cherry studio需要调用的llm有工具类型的时候才可以调用MCP。

  • Cline使用的是system prompt来实现，这个就无所谓用什么模型了。

下面我们以调用高德MCP为例，来说明两者的区别。我们的prompt就用浦东机场到人民广场怎么走？

Cherry Studio调用

  1. 第一个请求：在tool里面传入MCP的所有tool

  2. 第一个请求的返回：会返回Function的调用，可以返回多个

  3. 使用返回的参数，调用MCP并获取MCP的结果

  4. 发送MCP的结果给LLM：llm会结合MCP的结果给出答案

Cline调用

  1. cline一共使用了4次API完成输出：

  2. 第一次请求：cline会把可以调用的MCP在system prompt进行描述，它的system prompt很长，里面还会有cline可以调用的Tool，比如运行命令、读取文件等，调用MCP也算一个Tool

    1. 返回：
      1. cline的system prompt里面写了每次只能调用一个tool
        1. use_mcp_tool这些也是在system prompt里面定义好的

  3. cline调用MCP，获取返回结果。结果发送给llm

  4. 因为cline每次只会调用一个tool，所以要调用多次llm

  5. 任务完成的那次标记，这个也是写在system prompt里面的

Cline的system prompt

  • cline的system prompt很长，但是也可以分为几块
    • TOOL USE：cline自己调用的tool，有execute_command、read_file、use_mcp_tool、attempt_completion等。
    • MCP SERVERS
    • EDITING FILES
    • ACT MODE V.S. PLAN MODE
    • CAPABILITIES
    • RULES
    • SYSTEM INFORMATION
    • OBJECTIVE
    • USER’S CUSTOM INSTRUCTIONS
